```markdown
# 🎂 Lucchese Bot

Многофункциональный Discord-бот для управления сообществом. Включает системы дней рождений, контрактов, жалоб, оплат, ответственных и профилей.

## ✨ Основные возможности

### 🎂 Дни рождения
- Панель со списком дней рождений.
- Добавление / удаление записей через панель или команды.
- Автоматические напоминания за заданное количество дней (настраивается).
- Сортировка: по календарю или по ближайшему дню рождения.
- Автоудаление записи при выходе пользователя с сервера.

### 📜 Контракты
- Панель с правилами контрактов (редактируется через модальное окно).
- Управление набором: открытие / закрытие, запись участников.
- Автоматическая выдача роли участникам.
- Система заявок в контракт.
- Логирование действий в отдельный канал.

### 📩 Жалобы и предложения
- Анонимная подача жалоб и предложений через модальные окна.
- Панель с кнопками для быстрого доступа.
- Обработка заявок с комментариями.
- Отдельный канал для обращений (настраивается).

### 🏠 Оплата (дом/квартира и лаборатория)
- Панели для ввода данных об оплате.
- Автоматический расчёт оставшихся дней.
- Напоминания за N дней до окончания срока.
- Админ-панель: просмотр, редактирование, удаление записей, выгрузка БД.
- Отдельные каналы для логов оплаты.

### 👥 Ответственные роли
- Назначение / снятие ролей ответственности.
- Привязка к каналам (выдача прав на просмотр/общение).
- Автоматическая выдача/снятие общей роли ответственного.
- Таблица ответственных для просмотра.
- Выгрузка списка в CSV.

### 👤 Профили пользователей
- Регистрация профиля (никнейм + номер паспорта).
- Изменение профиля.
- Одобрение / отклонение профилей администраторами.
- Автоматическая выдача ролей и смена никнейма при одобрении.
- Отдельный канал для логов профилей.

### ⚙️ Настройки
- Единая панель управления с кнопками для всех настроек.
- Установка каналов логов (основной, контрактов, оплаты, профилей).
- Установка дней напоминаний (дни рождения и оплата).
- Установка ролей (ответственный за контракты, участники, техподдержка).
- Установка каналов (заявки, набор, техподдержка, жалобы).
- Смена статуса и активности бота.
- Сброс всех привязок каналов и ролей одной кнопкой.

### 📨 Отправка сообщений
- Команда `/say` для отправки сообщений от имени бота с заголовком, упоминаниями, изображениями и **отложенным запуском** (по расписанию).
- Просмотр и отмена запланированных сообщений.

### 🛠 Техподдержка
- Отдельный канал для заявок в техподдержку.
- Упоминание роли при создании заявки.

### 📋 Логирование
- Все действия логируются в настроенные каналы.
- Отдельные каналы для: общих логов, контрактов, оплаты, профилей.

### 🤖 Telegram-бот
- Синхронизация с Discord-ботом через общую БД.
- Управление днями рождения и напоминаниями через Telegram.

## 🚀 Установка и запуск

### Требования
- Python 3.8+
- Discord Bot Token
- (Опционально) Telegram Bot Token

### Установка
1. Клонируйте репозиторий:
```bash
git clone https://github.com/Dias101Paranoich/Lucchese_Bot.git
cd Lucchese_Bot
```

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Создайте файл `.env` на основе `.env.example`:
```env
DISCORD_TOKEN=ваш_токен
CREATOR_ID=ваш_discord_id
TRUSTED_USERS=id1,id2,...
# ... остальные переменные
```

4. Запустите бота:
```bash
python bot.py
```

### Telegram-бот (опционально)
Для запуска Telegram-бота выполните:
```bash
python telegram_bot.py
```

## 📁 Структура проекта

| Файл | Описание |
|------|----------|
| `bot.py` | Основной класс бота, задачи (tasks), обработка событий. |
| `commands.py` | Все слеш-команды. |
| `views.py` | Все интерактивные компоненты (кнопки, выпадающие списки, модальные окна). |

| `birthdays.py` | Логика дней рождений (расчёт дней, сортировка, создание embed). |

| `complaints.py` | Логика жалоб и предложений. |

| `house_payment.py` | Логика оплаты (SQLite, расчёт, экспорт). |
| `responsibles.py` | Логика ответственных ролей (SQLite, выдача/снятие). |
| `profiles.py` | Логика профилей (SQLite, одобрение). |
| `modals.py` | Все модальные окна. |

## 🔒 Права доступа

- **Создатель** (`CREATOR_ID`) — полный доступ ко всем командам и настройкам.
- **Доверенные пользователи** (`TRUSTED_USERS`) — могут добавлять/удалять дни рождения, управлять контрактами, жалобами, панелями.
- **Обычные пользователи** — просмотр панелей, отправка жалоб, регистрация профиля.

## 📦 Хранение данных

- **JSON** — дни рождения, состояния панелей, настройки, запланированные сообщения.
- **SQLite** — оплата (`house_payments.db`), ответственные (`responsibles.db`), профили (`profiles.db`).
- Все данные хранятся локально в папке с ботом.
