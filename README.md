## 📄 Обновлённый README.md

```markdown
# 🦾 Lucchese Bot

Многофункциональный Discord-бот для автоматизации дней рождений, управления контрактами (ролевыми наборами), отправки новостей, планирования сообщений, анонимных жалоб и **системы профилей**. Разработан с учётом потребностей игровых сообществ и не только.

---

## ✨ Особенности

### 🎂 Дни рождения
- Добавление/удаление дат рождения пользователей.
- Автоматические напоминания за настраиваемое количество дней (например, 7, 1, 0).
- Интерактивная панель со списком и сортировкой (по ближайшему или календарю).
- Автоудаление записей при выходе участника с сервера.

### 📜 Контракты (ролевые наборы)
- Создание правил контракта в отдельной панели.
- Отдельные каналы для заявок и объявлений набора.
- Роли ответственного (может управлять) и участников (могут записываться).
- Запись/отмена записи через кнопки, управление участниками (удаление ответственным).
- Автоматическое удаление устаревших сообщений (через 24 часа).
- Отдельный канал для логов действий с контрактами.

### 📰 Новости
- Отправка новостей в заданный канал с поддержкой изображений и упоминания роли.
- Команда `/news` для доверенных пользователей.

### ⏰ Планировщик сообщений
- Команда `/say` с возможностью указать время отправки (ДД.ММ.ГГГГ ЧЧ:ММ МСК).
- Поддержка упоминаний пользователей и ролей, прикрепление изображений.
- Просмотр и отмена запланированных сообщений.

### 📩 Анонимные жалобы и техподдержка
- Создание панели с кнопкой «Подать жалобу» (команда `/create_complaint_panel`).
- Отдельный канал для приёма жалоб (настраивается командами `/set_complaint_channel` и `/unset_complaint_channel`).
- При нажатии на кнопку открывается модальное окно с полями «Тема» и «Описание» – жалоба отправляется анонимно в заданный канал.
- Отдельная кнопка «🛠 Тех. поддержка» для отправки заявок в канал техподдержки с упоминанием роли.
- Логирование действий с жалобами в общий канал логов.

### 👤 Система профилей (НОВОЕ в v3.0)
- Регистрация профиля с указанием никнейма и номера паспорта.
- Автоматическая отправка заявки на одобрение в специальный канал.
- Упоминание нескольких ролей при новой заявке (настраивается в `.env`).
- Одобрение/отклонение профиля через кнопки с изменением цвета embed.
- Автоматическая смена никнейма пользователя на сервере при одобрении.
- Выдача ролей при одобрении (настраивается в `.env`).
- Отдельный канал логов для всех действий с профилями.
- Пользовательская панель (`/profile_panel`) и админ-панель (`/profile_admin_panel`).

### 🏠 Система оплаты (дом/квартира и лаборатория)
- Ввод данных об оплате (игровой ник, паспорт, VIP-статус, количество дней).
- Проверка статуса оплаты и просмотр профиля.
- Автоматические напоминания о скором окончании оплаты.
- Админ-панель для управления данными всех пользователей.
- Выгрузка базы данных в ZIP-архив.

### 👑 Система ответственных
- Назначение и снятие ответственных ролей.
- Автоматическая выдача доступа к привязанным каналам.
- Общая роль ответственных для всех назначенных пользователей.
- Таблица ответственных для просмотра.

### ⚙️ Управление ботом
- Панель настроек с кнопками для установки каналов логов, новостей, жалоб, дней напоминаний, ролей контрактов и т.д.
- Логирование действий в отдельный канал (уровни INFO, SUCCESS, WARNING, ERROR).
- Гибкая система прав: создатель (полный доступ), доверенные пользователи (ограниченный доступ).

### 🔒 Конфиденциальность
- Все данные хранятся локально в JSON-файлах и SQLite базах данных.
- Команды `/terms` и `/privacy` для отображения условий использования.

## 🚀 Установка и запуск

1. **Клонируйте репозиторий**
```bash
git clone https://github.com/dias101paranoich/lucchese-bot.git
cd lucchese-bot
```

2. **Создайте виртуальное окружение (рекомендуется)**
```bash
python -m venv venv
source venv/bin/activate  # Linux/Mac
venv\Scripts\activate     # Windows
```

3. **Установите зависимости**
```bash
pip install -r requirements.txt
```

4. **Настройте переменные окружения**
Скопируйте файл `.env.example` в `.env` и отредактируйте его (см. раздел **Настройка .env**).

5. **Запустите бота**
```bash
python bot.py
```

## 📁 Структура файлов данных

Все данные хранятся в текущей директории в формате JSON и SQLite:

**Основные:**
- `birthdays.json` – дни рождения пользователей.
- `panel_state.json` – состояние панели дней рождений (ID канала/сообщения).
- `reminder_config.json` – дни для напоминаний (например, `[7,1]`).
- `reminder_messages.json` – история отправленных напоминаний для автоудаления.
- `log_config.json` – ID канала логов.
- `scheduled_messages.json` – запланированные сообщения.
- `temp_images/` – временное хранилище изображений для отложенных сообщений.

**Контракты:**
- `contract_rules.json` – текст правил контрактов.
- `contract_panel_state.json` – состояние панели правил.
- `contract_config.json` – настройки контрактов (роли, каналы, мин. участников).
- `contract_session.json` – текущая сессия набора.
- `contract_log_config.json` – ID канала логов контрактов.
- `contract_manage_panel_state.json` – состояние панели управления набором.

**Жалобы:**
- `complaint_config.json` – ID канала для приёма жалоб.
- `complaint_panel_state.json` – состояние панели жалоб.
- `support_config.json` – настройки техподдержки (канал, роль).

**Оплата:**
- `house_payments.db` – SQLite база данных оплат.
- `house_payment_reminder.json` – дни напоминаний об оплате.
- `house_payment_log_config.json` – ID канала логов оплаты.

**Профили (НОВОЕ в v3.0):**
- `profiles.db` – SQLite база данных профилей.
- `profile_log_config.json` – ID канала логов профилей.
- `profile_user_panel_state.json` – состояние пользовательской панели профилей.
- `profile_admin_panel_state.json` – состояние админ-панели профилей.

**Ответственные:**
- `responsibles.db` – SQLite база данных назначений.
- `responsible_panel_state.json` – состояние панели управления ответственными.
- `responsible_table_state.json` – состояние таблицы ответственных.
- `responsible_role_config.json` – привязка ключей к ID ролей.
- `responsible_channels.json` – привязка ключей к ID каналов.

**Прочее:**
- `settings_panel_state.json` – состояние панели настроек.
- `birthday_sort_mode.json` – текущий режим сортировки дней рождений (`next`/`calendar`).
- `profile_log_config.json` – ID канала логов профилей.

## 📋 Команды бота

### 🎂 Дни рождения

| Команда | Описание | Доступ |
|---------|----------|--------|
| `/create_panel` | Создать интерактивную панель с днями рождения | только создатель |
| `/add_birthday <user> <date>` | Добавить день рождения пользователю (ДД.ММ) | доверенные |
| `/remove_birthday <user>` | Удалить запись | доверенные |
| `/refresh_panel` | Обновить панель | доверенные |
| `/restore_panel` | Восстановить панель после удаления | только создатель |

### 📜 Контракты

| Команда | Описание | Доступ |
|---------|----------|--------|
| `/create_contract_panel` | Создать панель с правилами контрактов | только создатель |
| `/create_contract_manage_panel` | Создать панель управления набором | только создатель |

**В панели управления:**
- **Открыть набор** – создаёт сообщение для записи в канале набора | ответственный
- **Закрыть набор** – закрывает набор (проверяет минимум участников) | ответственный
- **Заявка в Контракт** – отправляет заявку в канал заявок | участник
- **Управление участниками** – позволяет ответственному удалить участника | ответственный
- **➕ Добавить участника** – вручную добавить участника в набор | ответственный
- **Удалить набор** – принудительно удаляет набор (без проверки) | только создатель

**В сообщении набора:**
- **Записаться** – запись в контракт | участник
- **➕ Добавить в набор / Открыть набор** – принимает заявку и добавляет пользователя | ответственный

# 🎉 Lucchese Bot v3.0

## 🔥 Что нового

### 👤 Система профилей
- **Регистрация профиля** – пользователи могут заполнить форму с никнеймом и номером паспорта.

- **Админ-панель** – `/profile_admin_panel` для управления всеми профилями.

### 🔧 Улучшения и исправления
- **Исправлена ошибка `AttributeError`** в выпадающих списках – теперь корректно отображаются никнеймы и теги Discord.
- **Улучшен интерфейс одобрения** – при одобрении/отклонении цвет embed меняется, статус обновляется.
- **Обработка ошибок смены ника** – если бот не может изменить никнейм, процесс одобрения не прерывается, ошибка логируется.
